以下のようなリストのページを作成する。
![完成予想](/create.jpeg)

### 機能
- 検索(バリデーション付き)
- ページネーション
- バーチャライゼーション(意味を調べる)

その他できたらやる
・ログインログアウト
・登録、パスワード忘れ、メルアド変更
・エラートースト、確認モーダル
・各種保存時バリデーション
・お知らせ
・お気に入り
・ファイルアップロード、ダウンロード
・ドラッグアンドドロップでの並び替え、カスタマイズ

### 方向性
- dockerコンテナ上で作成。
- バックエンドはnode.jsを使用する。REST-APIにする。
    - 全てのユーザーを取得
    - 指定した番号から指定した番号までのユーザーを取得
    - 新しいユーザーの作成
    - ユーザー情報の変更
    - ユーザーの削除
- フロントは未定。jsを使って作成する。
    データ数は1000件程度を想定し、ページロード時に全てのデータを読み込み、操作に応じてJavaScriptで検索・絞り込み・ページネイションを行う。
    
    >開発段階では同一マシン上にクライアント・サーバ・DBがあるので10000件のデータ取得も時間はかからないが、実際のインターネット上で行うとどの程度時間がかかるか解らない。
    >いっぺんにバックエンドからデータを取得してクライアント側で(検索・絞り込み・ページネイションなどの)操作に合わせてデータを操作する方が良いのか、クライアント側の操作に合わせてバックエンドから取得する方が良いのかは現状解らないので調べるなり試すなりが必要。
